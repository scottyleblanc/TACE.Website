# tacedata-site

Personal portfolio and professional development site for [tacedata.ca](https://tacedata.ca).

## Purpose

This repository is the source for the tacedata.ca website. It serves as:

- A portfolio of technical projects (PowerShell automation, Oracle, AWS)
- A record of professional evolution — DBA to automation engineer to cloud-capable practitioner
- A blog for technical writing, published to LinkedIn for broader reach
- A place to share honest observations on the use of AI in real engineering work

This is not a consulting brochure. It is a working journal with proof of work.

## Repository Structure

```
tacedata-site/
├── README.md               ← this file
├── docs/
│   ├── TODO.md             ← active build checklist
│   ├── STAGES.md           ← phase definitions and current state
│   └── DECISIONS.md        ← key decisions and rationale (created as needed)
└── hugo-eval/              ← Hugo evaluation workspace (Stage 1)
    ├── hugo.toml           ← Hugo site config (created during eval)
    ├── content/
    │   ├── posts/          ← blog post Markdown files
    │   └── projects/       ← project write-up Markdown files
    ├── themes/             ← theme(s) under evaluation
    ├── static/             ← static assets (images, files)
    └── layouts/            ← custom layout overrides (if needed)
```

Once a theme and workflow are confirmed, `hugo-eval/` graduates to the production
site root and is renamed accordingly.

## Local Development

### Prerequisites

- Hugo Extended (`winget install Hugo.Hugo.Extended`)
- Git

### Run locally

```bash
cd hugo-eval
hugo server -D
```

Browse to `http://localhost:1313`. The `-D` flag includes draft content.

### Build

```bash
cd hugo-eval
hugo
```

Output is written to `hugo-eval/public/`. This directory is not committed — it is
generated by the CI/CD pipeline on deploy.
