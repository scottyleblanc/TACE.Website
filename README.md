# tacedata-site

Personal portfolio and professional development site for [tacedata.ca](https://tacedata.ca).

## Purpose

This repository is the source for the tacedata.ca website. It serves as:

- A portfolio of technical projects (PowerShell automation, Oracle, AWS)
- A record of professional evolution — DBA to automation engineer to cloud-capable practitioner
- A blog for technical writing, published to LinkedIn for broader reach
- A place to share honest observations on the use of AI in real engineering work

This is not a consulting brochure. It is a working journal with proof of work.

## Local Development

### Prerequisites

- Hugo Extended v0.158+ (`winget install Hugo.Hugo.Extended`)
- Git

### Run locally

```powershell
cd hugo-eval
hugo server -D
```

Browse to `http://localhost:1313`. The `-D` flag includes draft content.

### Build

```powershell
cd hugo-eval
hugo --minify
```

Output is written to `hugo-eval/public/`. This directory is not committed — it is
generated by the CI/CD pipeline on deploy.

## Deployment

Push to `main` triggers the GitHub Actions pipeline automatically.

### Pipeline steps

1. Checkout repo with submodules (Blowfish theme)
2. Build Hugo Extended with `--minify`
3. Authenticate to AWS via OIDC (no long-lived credentials)
4. Sync `hugo-eval/public/` to `s3://tacedata-s3-bucket-02/tacedata-site/`
5. Invalidate CloudFront distribution to serve updated content

### AWS resources

| Resource | Value |
|---|---|
| S3 bucket | `tacedata-s3-bucket-02` (prefix: `tacedata-site/`) |
| CloudFront distribution | `E1144JDZXTVLFW` |
| CloudFront domain | `d2i8qth3t7mbtl.cloudfront.net` |
| IAM role | `tacedata-github-deploy` |
| Region | `ca-central-1` |

### GitHub secrets required

| Secret | Purpose |
|---|---|
| `AWS_ROLE_ARN` | IAM role ARN for OIDC authentication |

See `config/runbook-stage3-aws.md` for full infrastructure setup commands.
